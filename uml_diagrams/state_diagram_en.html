CubitStatuses Lifecycle

        stateDiagram-v2
            direction LR
            [*] --> init: Initial State

            init --> loading: fetchData() is called
            init --> noLoading: fetchData() is called (with existing cache)

            loading --> done: API call succeeds
            loading --> error: API call fails

            noLoading --> done: Background fetch succeeds
            noLoading --> error: Background fetch fails

            done --> loading: refreshData() is called
            error --> loading: retry() is called

            note right of init
                The Cubit starts in the 'init' state.
            end note

            note right of loading
                The 'loading' state indicates a mandatory data fetch is in progress.
                The 'noLoading' state is similar but occurs when stale cache is
                displayed while a background fetch happens.
            end note
